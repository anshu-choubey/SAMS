🚨 THE PROBLEM
══════════════
User B shows live face → passes liveness ✅
User B shows User A's photo → passes recognition ✅
Result: User B marks attendance for User A 🔓 FRAUD

✅ THE SOLUTION
═══════════════
Both checks happen on THE SAME camera frame
5 consecutive frames must pass both checks
Confidence stability monitored to detect photo swaps
Enhanced anti-photo detection measures

🧪 HOW TO TEST
══════════════
1. Normal Flow:
   - Mark attendance normally
   - Should take 2-3 seconds
   - Shows progress: X/5 frames

2. Photo Attack Test:
   - Show printed photo
   - Should fail with liveness error
   - Error: "No natural head movement" or "Low texture complexity"

3. Photo Swap Test:
   - Show live face briefly
   - Switch to photo quickly
   - Should fail with "Suspicious: Face match changed suddenly"
